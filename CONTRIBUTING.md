# Contributing to NuraLife

Thanks for contributing to NuraLife.

## Branching

- Create focused branches from `work`.
- Keep changes scoped to a single purpose.

## Commit convention

Use Conventional Commits when possible:

- `feat:` New feature
- `fix:` Bug fix
- `docs:` Documentation only changes
- `chore:` Tooling/maintenance
- `refactor:` Internal code improvements
- `test:` Test-related changes

## Pull request checklist

Before opening a PR:

- [ ] Run `bash scripts/ci/checks.sh`
- [ ] Update documentation when behavior or process changes
- [ ] Keep PR title clear and imperative
- [ ] Include rationale and impact in PR description

## Coding standards

- Prefer small, composable units.
- Keep naming explicit and intention-revealing.
- Avoid adding dependencies without a clear need.
